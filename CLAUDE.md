For all Python-related work:

Before running, suggesting, or generating any Python commands:
1. Check whether a virtual environment is active (e.g., $VIRTUAL_ENV).
2. If none exists:
   - For backend API work: create one at backend/.venv using `cd backend && python3 -m venv .venv`
   - For scripts work: use the backend venv or create scripts/.venv if separate dependencies needed
3. Activate the virtual environment before proceeding.
4. Never install Python packages globally.
5. If a virtual environment is not active, stop and create/activate it first.
